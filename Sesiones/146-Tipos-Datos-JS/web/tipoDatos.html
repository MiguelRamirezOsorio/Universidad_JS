Tipo de datos en JS:

Entre los tipos de datos basicos en JavaScript estan los siguientes:

  • Tipo String: llamados tambien cadenas; estas almacenan los datos de un grupo de caracteres, ademas este tipo de dato debe ir entre comillas.
    Ejm:
  • Tipo numerico: estas alamcenan numeros desde enteros hasta decimales, este tipo de datos esta asignado medainte el var.
    Ejm:
  • Tipo Object: son los que almacenan un conjunto de atributos de un objeto como por ejemplo el nombre, apellido, email, telefono de una persona, entre otros. Estos se almacenaran como un array del objeto señalado.
    Ejm:
  • Tipo Boolean: es aquel que puede representar valores de lógica binaria, esto en general son los 2 valores que representan verdadero o falso. Mayormente se usan para ampliar, limitar o definir funciones, variables, entre otros.
    Ejm:
  • Tipo Function: En términos generales, una función es un "subprograma" que puede ser llamado por código externo (o interno en caso de recursión) a la función. Al igual que el programa en sí mismo, una función se compone de una secuencia de declaraciones, que conforman el llamado cuerpo de la función. Se pueden pasar valores a una función, y la función puede devolver un valor.
    En JavaScript, las funciones son objetos de primera clase, es decir, son objetos y se pueden manipular y transmitir al igual que cualquier otro objeto.
    Ejm:
  • Tipo Symbol: es un tipo de dato cuyos valores son únicos e immutables. Dichos valores pueden ser utilizados como identificadores (claves) de las propiedades de los objetos. Cada valor del tipo Symbol tiene asociado un valor del tipo String o Undefined que sirve únicamente como descripción del símbolo.
    Ejm:
  • Tipo Clase: son una mejora sintáctica sobre la herencia basada en prototipos de JavaScript. La sintaxis de las clases no introduce un nuevo modelo de herencia orientada a objetos en JavaScript. Las clases de JavaScript proveen una sintaxis mucho más clara y simple para crear objetos y lidiar con la herencia. En si una clase tiene como tipo de dato al function, siendo como una opcion de este.
    Ejm:
  • Tipo Undefined: Es uno de los valores primitivos de JavaScript. es una propiedad del objeto global, es decir, una variable de alcance global. El valor inicial de undefined es el valor primitivo undefined.
    Se declara como undefined a la variable que no se le ha asignado valor, o no se ha declarado en absoluto (no se declara, no existe). Un método o sentencia también devuelve undefined si la variable que se está evaluando no tiene asignado un valor. Una función devuelve undefined si no se ha devuelto un valor.
    Ejm:
  • Tipo Null: es un literal de Javascript que representa intencionalmente un valor nulo o "vacío". En este caso null es un literal definido en la especificación del lenguaje, no una variable global como en el caso undefined, por lo tanto, es una palabra reservada y no podremos utilizarla como nombre de variable. A diferencia de undefined, null no dispone de un tipo específico y en el caso de preguntar con typeof sobre un elemento con valor null nos dirá que es un objeto, lo cual puede desconcertarnos, ya que en la práctica no podemos tratarlo como al resto de objetos.
    Ejm:
  • Tipo Array: Son un conjunto de datos ordenados por posiciones y todos asociados en una sola variable. Los datos pueden ser de cualquier tipo de dato, es decir, es posible crear una array que tenga una cadena en la primera posición, un número en el segundo, un objeto en el tercero y así sucesivamente. Podremos acceder a estos distintos datos de manera independiente o agrupándolos. Cabe resaltar que un array de tipo objeto.
    Ejm:

Valor vacio empty String:

Para el caso de un String que no contenga ningun valor y solo comillas ("") existe el mensaje empty String, es asi como se pueden asignar cadenas vacias a nuestras variables. Su tipo de dato es String.

Uso de var, let, const:

Se aplicara cada uno cuando:

  • var: Es una variable que SI puede cambiar su valor y su scope es tanto local como global.
  • let:Es una variable que también podra cambiar su valor, pero solo vivirá(Funcionara) en el bloque donde fue declarado.
  • const: Es una constante no reasignable la cual NO cambiara su valor en ningún momento en el futuro.
